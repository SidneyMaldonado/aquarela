1.	Os testes devem usar autenticação JWT (Bearer token) para os endpoints protegidos das controllers Contas, Dividas e Usuarios, ou devem remover a autenticação nos testes?
Podem remover nos testes
2.	Cada endpoint deve ter apenas um teste de sucesso ou também testes de cenários de erro (NotFound, BadRequest, etc)?
Também deve ter testes de cenários de erros
3.	Os testes devem seguir o mesmo padrão do LoginEndToEndTest usando uma factory compartilhada ou cada controller deve ter sua própria factory?
Cada controller deve ter sua própria factory para garantir isolamento entre os testes e evitar efeitos colaterais.
4.	Os testes devem ser organizados em classes separadas para cada controller ou podem ser agrupados em uma única classe de testes?
Devem ser organizados em classes separadas para cada controller, seguindo o princípio de responsabilidade única e facilitando a manutenção dos testes.
